Ocean Drive V2
Ocean Drive V2 is a Fivem roleplay server that is being developed by a that team exists out of a variety amount of talented people.

FiveM is a modification for Grand Theft Auto V enabling you to play and create customized multiplayer servers.

The things that will make our server stand out among the other servers is that almost all of the scripts will be custom made. This will give us full control on what is happening, easily extend, deploy new features on the fly and maintain the server more easily which is really important to us. Also the community matters a lot to us because with out them we would not be doing this. We listen closely to our community and develop scripts based on their needs.

Our goal is to become the biggest roleplaying server among the Fivem community. We know this is a very ambitious goal but I believe it is possible due to our talented team.

Ocean Drive V2 will be released somewhere in 2021 to the public. Date: 2020 & present

Ocean drive already has already one roleplaying server which is known as Ocean Drive V1. This server exists out of community scripts and scripts that are edited by some of the developers in our team. The issue with this server is that the scripts don’t really work well together or the scripts miss features and are programmed poorly which leads to low fps on the client side.

That is why I started working on Ocean Drive V2. Currently it is just me working on V2 while the other people in the team maintain V1. I also have my cousin help me build V2 because he wants to learn how to program and work with databases. So I let him make some scripts and give him some tips on how to approach making those scripts and afterwards I will review his code and give him feedback.

After the main functionality is implemented in V2 the entire team will help me continue build the server. The main things that need to be done before the team can help me work on V2 are:
  • Code documentation
  • Guides for scripts
  • Database
  • Pipelining for the server
  • Some dev tools like flying around, teleporting etc.
  • Notifications
  • Chat system
  • Permissions system
  • Basic commands for kick, ban, grant [permission] etc.
  • Character creation
  • Character select
  • Shops – clothing, tattoo, food, pawn etc.
  • Inventory + trading + items
  • Phone + app system
  • Job system
  • Jobs - Police, doctor, garbage collector, delivery person etc.
  • Heists
  • Create races
  • Car dealer
  • Car customization
  • Custom key binding
  • Setup Tokovoip (for talking in game)
This is almost the complete list of all the high priority tasks. These need to be finished before we can release V2.

We work with scrum but do not apply the full methodology. We do this through Gitlab because with Gitlab you get free pipeline minutes and there is a tool for keeping track of issues and assign those issues too developers in the team.

So the way our Gitlab is structured is we have 2 branches and feature branches. First branch is the master branch which reflects the running version of the server. Second branch is the development branch, this is where features will be merged once they are finished. After a bunch of new features are in the development branch we have some community members test them and if everything seems good it will be merged with the master branch. Also all of the merge request will be reviewed by a team member to avoid bugs.

There are many people in our team with different nationalities and different roles in the team:
  • 3 people for support. They are most of the time the first to respond to tickets, manage the Discord and are active players in the server with rights like kick and ban.
  • 4 hobbyist developers
  • 2 developers with a degree in programming
  • 1 artist
  • 1 community manager
  • 1 social media manager
